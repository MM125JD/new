**图片懒加载（Lazy Loading）** 的核心作用用四个字概括就是：**延迟加载**。

简单来说，当用户打开一个网页时，浏览器默认会**一次性下载**页面上所有的图片，不管这些图片是在屏幕顶端，还是藏在要滚动好几屏才能看到的页面最底部。

开启了懒加载后，游戏规则就变了：**只有当用户向下滚动网页，图片即将进入屏幕视口（可视区域）时，浏览器才会真正开始下载这张图片。**

这个功能具体能带来以下几个核心好处：

### 1. 大幅提升“首屏”加载速度

如果你的 Markdown 文档很长，里面插了几十张高清截图或架构图。

- **不开启懒加载**：用户一打开网页，浏览器要同时下载几十张图片，网络带宽被占满，导致网页卡顿、白屏，用户体验很差。
    
- **开启懒加载**：用户刚打开网页时，浏览器只下载最顶端能看到的 1~2 张图片，网页瞬间就刷出来了（首屏秒开）。
    

### 2. 节省用户的手机流量（非常重要）

如果一个用户用手机流量打开了你的 VitePress 文档，他可能只想看第一段的文字，看完就直接关掉网页了。

- **不开启懒加载**：虽然他只看了第一页，但手机在后台已经悄悄把他后面没看过的几十张图片全部下载完了，白白浪费了用户的流量。
    
- **开启懒加载**：他没看过的图片，网络请求连发都不会发，帮用户省下了流量。
    

### 3. 减轻你服务器的压力（降低带宽成本）

所有的图片都是托管在服务器（或者 CDN）上的。每次用户访问，服务器都要传输图片数据。懒加载减少了那些“用户没看完网页就关闭”所导致的无效图片请求，能直接**降低你服务器的带宽消耗和流量成本**。

### 🛠️ 怎么在 VitePress 中开启？

就像你提到的，在 VitePress 中开启它极其简单，只需要在你的项目配置文件（如 `.vitepress/config.mts` 或 `.vitepress/config.js`）中加上一行 `lazyLoading: true`：

TypeScript

```mts{8}
// .vitepress/config.mts
export default {
  title: "我的文档",
  themeConfig: {
    // ... 其他配置
  },
  markdown: {
    lazyLoading: true // 开启 Markdown 图片懒加载
  }
}
```

开启后，VitePress 在把你的 Markdown 编译成网页时，会自动给所有的 `<img>` 标签加上 `loading="lazy"` 这个标准的浏览器原生属性，不需要你手动去写复杂的 JavaScript 脚本。

你目前的文档里是有非常多的图片需要展示，还是在提前规划网站的性能优化呢？

::: question
lazyLoading: true这串代码为什么添加到markdown括号内部
::::
这是因为 VitePress 的配置文件有一个清晰的**分层管理架构**。

在 VitePress 中，`markdown` 是一个**独立的配置大类（对象）**，所有专门用来控制 Markdown 文件如何解析、编译、渲染的开关，都必须统一打包塞进 `markdown: { ... }` 的大括号内部。

如果把它拆开来看，它的逻辑是这样的：

TypeScript

```
export default {
  // 1. 这里放全局的基础配置
  title: "我的文档",
  description: "这是一个测试网站",

  // 2. 这里放和“网站主题、导航栏、侧边栏”相关的配置
  themeConfig: {
    nav: [ { text: '首页', link: '/' } ],
    sidebar: [ ... ]
  },

  // 3. 这里专门放和“Markdown 文本解析”相关的配置
  markdown: {
    lineNumbers: true,   // 比如：是否在代码块显示行号
    lazyLoading: true    // 比如：是否给 Markdown 里的图片开启懒加载
  }
}
```

### 为什么不能直接写在外层？

如果把 `lazyLoading: true` 直接写在最外层（和 `title` 平级），VitePress 的底层解析器就会“迷路”。

因为最外层是留给整个网站全局参数的，VitePress 根本不知道这个 `lazyLoading` 到底是要给网站的哪个功能用（是给系统里的某些插件用？还是给主题渲染用？）。

只有把它放进 `markdown: { ... }` 内部，VitePress 才会明白：“**哦！原来这是针对 Markdown 编译器的指令。当你在把 `.md` 文件转换成 HTML 网页时，记得顺便把里面的图片都加上懒加载属性。**”

这就好比归类整理物品：衣服放进衣柜，碗筷放进碗柜。`lazyLoading` 属于 Markdown 处理规则的一员，所以必须老老实实待在 `markdown` 的大括号格子里。
