## 1.调整网络（临时调整）
  ### 终端输入
```
$env:http_proxy="http://127.0.0.1:10808"
$env:https_proxy="http://127.0.0.1:10808"
```
	- 注意这是临时调整一下，关闭命令行之后就不行了
## 2.问题2
$env:http_proxy="http://127.0.0.1:10808"
$env:https_proxy="http://127.0.0.1:10808"输入了这行命令以后以后直接运行openclaw gateway就行了吧，不需要重新输入上面这两行命令了


**不是的**，下次重新打开终端时，你**仍然需要重新输入**这两行命令。

因为在 PowerShell 中，使用 `$env:变量名` 设置的环境变量是**临时生效**的。它的生命周期有以下特点：

- **当前窗口有效**：这两行命令只对你**当前正在使用**的这一个 PowerShell 窗口生效。
    
- **关闭即失效**：一旦你把这个蓝黑色的终端窗口关掉了，或者电脑重启了，这个临时内存就会被直接清空。
    
- **新窗口不共享**：如果你同时打开了两个 PowerShell 窗口，在 A 窗口输入的代理命令，B 窗口是吃不到代理的。
    

### 💡 现在的正确操作：

1. 你现在在当前窗口输入完这两行命令后，**直接运行 `openclaw gateway` 是完全没问题的**。
    
2. 只要你**不关闭**这个窗口，在里面反复运行、停止、再运行 `openclaw gateway`，都不需要重新输入代理命令。
    
3. **但是**，如果你明天重新开机，或者不小心把这个终端窗口关了，再次打开新窗口准备运行时，就必须**先复制贴入这两行代理命令，然后再敲 `openclaw gateway`**。
    

### 🌟 嫌麻烦？教你一个“一劳永逸”的办法：

如果你不想每次开窗口都复制一遍，可以直接把代理**永久写入**你的 Windows 系统环境变量中（这样无论什么时候开窗口，程序都默认自带代理）：

1. 在当前 PowerShell 中执行下面这两行命令（注意：这里使用的是 `[Environment]::SetEnvironmentVariable` 语法）：
    
    PowerShell
    
    ```
    [Environment]::SetEnvironmentVariable("http_proxy", "http://127.0.0.1:10808", "User")
    ```
    
    PowerShell
    
    ```
    [Environment]::SetEnvironmentVariable("https_proxy", "http://127.0.0.1:10808", "User")
    ```
    
2. 执行完这两行后，**关闭当前窗口，重新打开一个新的 PowerShell**。
    
3. 以后你无论什么时候打开终端，直接输入 `openclaw gateway` 就能直接通网，再也不用手动去敲代理命令了！

# 二.如何创建web search密钥（如果你还没有创建密钥）
没问题，咱们重新走一遍流程！

因为 OpenClaw 底层实际上是通过 **Gemini API** 的通道来实现联网搜索功能的，所以我们需要创建的是一个**拥有 Gemini API 权限的万能密钥**。

请按照以下四步进行操作：

### 第一步：登录谷歌云控制台

1. 在浏览器中打开 **[Google Cloud 控制台凭据页面](https://console.cloud.google.com/apis/credentials)**。
    
2. 确保页面右上角登录的是你常用的谷歌账号。如果提示需要选择项目，选择你之前创建过的项目即可（如果没有项目，点击新建一个）。
    

### 第二步：生成新密钥

1. 在页面顶部，点击 **`＋ 创建凭据` (CREATE CREDENTIALS)** 按钮。
    
2. 在弹出的下拉菜单中，选择第一项：**`API 密钥` (API key)**。
    
3. 系统会自动弹出一个小窗口，生成一串长乱码（类似于 `AIzaSy...`）。**先点击右侧的复制图标把它保存下来**，这就是你要填入 `openclaw.json` 的密钥。然后点击“关闭”。
    

### 第三步：配置密钥权限（关键步骤）

刚建好的密钥默认是没有任何限制的，为了让它稳定用于 OpenClaw 的搜索，我们需要给它指定权限：

1. 在“API 密钥”列表中，找到你刚刚创建的那个新密钥（通常默认叫 _API 密钥 1_ 或 _API 密钥 2_），点击它右侧的**三个点**，选择 **`修改密钥` (Edit key)**。
    
2. 页面往下拉，找到 **“选择 API 限制” (API restrictions)** 这一栏。
    
3. 将选项从“不限制密钥”改为 **`限制密钥` (Restrict key)**。
    
4. 点击下方弹出的下拉选择框，在里面搜索或直接勾选 **`Gemini API`**。
    
5. 勾选完成后，点击下拉框右下角的 **`确定`**，然后点击页面最下方的 **`保存`**。
    

### 第四步：填入 OpenClaw

回到你的 `openclaw.json` 文件中，将刚刚复制的那串密钥粘贴到 `webSearch` 的配置里：

JSON

```
"google": {
  "enabled": true,
  "config": {
    "webSearch": {
      "apiKey": "你刚刚复制的以AIzaSy开头的密钥"
    }
  }
}
```

保存文件，再次去终端里启动 `openclaw gateway`，这时候你的智能体就能高高兴兴地带着这把钥匙去联网搜索了！
# 三 . 这两种密钥的区别
![[Pasted image 20260718211336.png]]
从你这张最新的控制台截图来看，这两个密钥在**本质权限（“限制”一列都写着 Gemini API）上其实没有区别**，但它们的**生成渠道和账号绑定机制**有很大的不同。

我来为你拆解这两种密钥的底层区别，以及为什么说第一个（或手动创建的）更适合拿来做 web search 这种自定义配置：

### 1. 它们的核心区别是什么？

我们可以从“Bound account（绑定账号）”这一列看出两者的根本差异：

- **`Gemini API Key`（第二个，带有关联账号）：自动生成的“托管密钥”**
    
    - **来源**：这个密钥通常不是你在 Google Cloud 控制台手动点出来的，而是你在使用 Google AI Studio 或者顶点 AI（Vertex AI）等官方专门的模型开发平台时，系统为你**自动托管生成**的。
        
    - **特点**：它深度绑定了一个名为 `ais-gemini-key-...` 的谷歌服务账号（Service Account）。这个账号就像是谷歌给 AI 服务开辟的“专车专用通道”。因为它的绑定关系很死，主要是为了纯粹的底层模型对话（GenerateContent）而设计的，如果你想魔改它的权限或者跨项目混用，经常会受到服务账号本身安全策略的限制。
        
- **`API 密钥 2`（第一个，没有关联账号）：你手动控制的“原生通用密钥”**
    
    - **来源**：这是你直接在 Google Cloud 控制台通过点击 “+ 创建凭据” **手动创建**出来的原生 API 密钥。
        
    - **特点**：它表现为一个干净的“——”（无绑定状态）。它不从属于任何自动化的 AI 托管服务，因此它的**控制权完全在你自己手里**。你可以自由地在后台修改它的名字、限制它的 IP 范围、或者给它解开/加上各种谷歌云服务的权限锁。
        

### 2. 为什么说第一个更适合作为专门用于 web search 的密钥？

这其实是由于 OpenClaw 的程序设计和调用逻辑决定的：

- **OpenClaw 的调用逻辑**：正如刚才的报错所示，OpenClaw 是把这个密钥直接当作**大模型原生能力（Gemini API）的通行证**传给程序的。
    
- **权限的纯净度与冲突隔离**：第二个密钥（Gemini API Key）因为绑定了官方的自动化服务账号，它的底层权限受制于谷歌的预设规则，你如果乱改它或者用来做多功能的开发，非常容易导致官方的 AI Studio 平台同步报错。而第一个密钥是**独立的、干净的**。你把它单独挑出来配置给 `openclaw.json`，即使以后需要在谷歌后台调整它的 API 限制（比如哪天 OpenClaw 升级了，真的需要直接调用 Custom Search），你也可以直接在第一个密钥上改，而**完全不会影响到你其他大模型项目的正常对话**。
    

### 总结

简单来说，第二个是“系统自动配的专用钥匙”，而第一个是你“自己配的万能钥匙”。**把第一个（API 密钥 2）单独作为 OpenClaw 的专属搜索密钥，是最安全、最不容易和别的 AI 项目撞车或发生权限冲突的做法。**

## 问题分析

| 项目 | 状态 |
|------|------|
| 网络连接 | ✅ 能连上 Google API 了 |
| API 密钥 | ✅ 密钥有效 |
| **API 权限** | ❌ 密钥没有启用 Gemini API |

---

## 解决方案

你需要去 **Google Cloud Console** 给这个 API 密钥启用权限：

1. 打开 [Google Cloud Console](https://console.cloud.google.com)
2. 找到 **API 和服务 → 凭据**
3. 找到这个 API 密钥
4. 启用 **Generative Language API**（Gemini API）

或者重新创建一个密钥，确保勾选 Gemini API 权限。
