如何把云端的内容推送到本地 ​

把云端内容同步到本地，在技术交流中，我们一般不叫“推送（Push）”，而是叫“拉取（Pull）”、“同步（Sync）”或者“下载（Download）”（因为 Push 通常指把本地代码送到云端）。不过这不重要，咱们对齐了意思就行！

取决于你现在在操作的是博客代码（Git）、Obsidian 笔记，还是多设备间的文件传输，实现方式会有些许不同。以下是几种最常见的场景和操作方法：

场景一：VitePress 博客代码（Git / GitHub） ​

如果你在 GitHub 或 Gitee 上托管了你的博客代码，并且在云端（比如在其他电脑上，或者在 GitHub 网页端）直接修改了文章，想要拿回现在的本地电脑：

1. 如果本地已经有这个项目文件夹 ​

打开终端（Terminal），先确保你在项目根目录下（比如 yangguangblog），然后运行：

Bash

git pull origin main

💡 解释：这行命令会去云端（origin）把最新的分支（比如 main 或 master）的代码“拉”下来，并自动与你本地的代码合并。

2. 如果换了新电脑，本地什么都没有 ​

你需要把云端的仓库整个“克隆”下来：

Bash

git clone <你的项目GitHub仓库地址>

场景二：Obsidian 笔记同步（云端 $\rightarrow$ 本地） ​

如果你是在手机上写了笔记，或者通过云端备份了 Obsidian 仓库，想同步到本地电脑上，有几种非常高效且保护隐私的本地化方案：

  • 方案 A：Syncthing（局域网/互联网多设备实时同步）

    • 特点：完全免费、开源，不需要经过第三方云服务器，直接在设备之间点对点传输。

    • 怎么做：在电脑和手机（或另一台设备）上同时安装 Syncthing，添加设备后，将本地的 Obsidian 文件夹设为共享，它就会在后台自动、双向地把云端/其他设备的内容同步到本地。

  • 方案 B：Git 自动同步

    • 在本地 Obsidian 安装插件 Obsidian Git。它可以设置每隔 X 分钟自动进行 git pull 和 git push，把托管在私有 Git 仓库里的笔记自动同步下来。

  • 方案 C：Remotely Save 插件

    • 如果你使用的是阿里云盘、腾讯云 COS、坚果云（WebDAV）等云存储，可以在 Obsidian 里安装 Remotely Save 插件，绑定后一键即可把云端内容同步到本地。

场景三：多设备/自动化脚本文件传输 ​

如果你在跑一些自动化脚本（比如多设备脚本），想把云端服务器（或多台设备）上的数据、日志自动下载到本地电脑，可以用以下方案：

  • 使用 SCP / SFTP 命令（适合服务器到本地）：

    Bash

    scp -r username@your_server_ip:/path/to/cloud/folder /local/path/to/save

    (这行命令会把云端服务器指定目录下的所有文件，安全地复制到你本地指定的路径下。)

  • 使用 Python / Kotlin 编写自动下载脚本：

    利用云服务商提供的 SDK（如腾讯云 COS、阿里云 OSS 等），写一个简单的定时任务，每天自动拉取最新的备份压缩包并解压到本地。

我不希望方案一是合并，而是完全替换 ​

简单来说，git clean -fd 的作用是：强力清除本地所有“未被 Git 追踪”的垃圾文件和文件夹。

在 Git 中，文件分为“已追踪（Tracked）”和“未追踪（Untracked）”两种。

  • 已追踪：你执行过 git add，或者之前已经提交（Commit）过的文件。

  • 未追踪：你新创建的、但从未 git add 过的文件/文件夹，或者编译、运行程序时自动生成的临时缓存。

git reset --hard 只能把已追踪的文件恢复到和云端一模一样的状态，但对于你本地那些未追踪的“孤儿”文件，它是无能为力的。这时候就需要配合 git clean -fd 来一次彻底的大扫除。

拆解这个命令的参数 ​

这个命令由三部分组成：

  • git clean：这是 Git 专门用来清理未追踪文件的命令。

  • -f (force)：强制执行。Git 为了防止你误删文件，默认是不允许直接 clean 的，必须加上 -f 告诉 Git：“我很清醒，强制帮我删掉”。

  • -d (directory)：连同文件夹一起删。如果不加 -d，Git 只会帮你删掉未追踪的孤儿_文件_，而那些未追踪的空_文件夹_还会残留在那里。加上 -d 就能连文件夹一起拔除。

⚠️ 避坑指南：执行它之前要注意什么？ ​

这个命令非常威猛，但也极具破坏性：

  1. 不可逆性：被这个命令删除的文件不会进入电脑的回收站/垃圾桶，它是直接从硬盘上彻底抹去的，无法找回。

  2. 防误删技巧：如果你不确定会删掉哪些文件，可以先运行下面这行命令进行“演习”：

    Bash

    git clean -fdn

    💡 多加了一个 -n (dry-run)，Git 就会列出一张清单，告诉你：“如果真动手，我会删掉哪些文件”，但它现在不会真的删。看完清单确认没问题了，再去掉 -n 执行真枪实弹的 git clean -fd。
